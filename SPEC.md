# 财务自由模拟评估系统 — 规格说明书 (SPEC)

## 1. 项目概述

开发一个基于 Web 的财务自由模拟评估工具。用户输入家庭资产、收入、支出、小孩抚养等信息，系统逐年模拟现金流直到夫妻双方均达到 85 岁，判断是否能实现财务自由，并展示详细的年度数据。

### 核心评估逻辑

系统从当前年份开始，逐年模拟至夫妻双方均到达 **目标年龄（默认 85 岁）** 的最后一年，可以画出一条模拟曲线。每一年的计算公式为：

```
年末净资产 = 年初净资产 + 当年总收入 - 当年总支出
```

其中：
- **当年总收入** = 工资收入（退休前）+ 房租收入 + 分红收入 + 其他收入，各项均按通胀率逐年增长
- **当年总支出** = 年化生活支出 + 小孩教育支出 + 意外预留分摊 + 负债年还款额，各项均按通胀率逐年增长
- **年初净资产** = 上年末净资产（首年为「总资产 - 总负债」）

> **财务自由判定条件**：若所有年份的年末净资产均 ≥ 0，则判定为 **财务自由**；否则输出 **最大资金缺口**（即所有年份中净资产的最小负值）。

| 大学阶段—年限 | 数字 | 7 | — |
| 大学阶段—每年费用 | 数字 | 100,000 | 该阶段每年教育费用 |

> 教育阶段共 4 个：学前（0-5岁）→ 小学（6-11岁）→ 中学（12-17岁）→ 大学（18-24岁）。系统根据小孩当前年龄，自动判断未来每年是否处于某个教育阶段。毕业赞助仅在最后一个阶段（大学）结束当年一次性支付。

#### 3.1.6 意外预留

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 意外预留总额（万元）| 数字 | 10 | 整个周期预留的意外备用金 |
| 分摊方式 | 下拉选择 | 一次性扣除 | 一次性扣除 / 逐年均摊 |

- **一次性扣除**：第一年从净资产中直接扣除该金额
- **逐年均摊**：从第一年到最终年，平均分摊到每年，算入当年总支出

#### 3.1.7 经济参数

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 年化通胀率 | 百分比 | 2% | 收入和支出每年按该比率增长 |
| 资产年化收益率 | 百分比 | 2% | 净资产每年产生的额外收益（如理财利息），默认与通胀率相同，采用保守策略 |

### 3.2 计算模块（Financial Freedom Calculator）

计算模块接收输入模块的所有数据，执行以下步骤：

**Step 1 — 确定模拟年限**
- 计算终年 = 当前年份 + (预期寿命 − 夫妻较年轻者年龄)
- 模拟从当前年份到终年共 N 年

**Step 2 — 逐年模拟**
- 对第 t 年（t = 0, 1, ..., N-1）：
  - 通胀系数 = (1 + 通胀率)^t
  - **当年总收入** = (工资收入×退休标记 + 房租+分红+其他) × 通胀系数
    - 退休标记 = 1 若当前年龄 < 退休年龄，否则 0
  - **当年总支出** = (年生活消费 + 小孩教育支出) × 通胀系数 + 意外预留年分摊额
    - 小孩教育支出：根据小孩当前年龄和 t 推算当年是否处于就学阶段
    - 意外预留：若为逐年均摊则每年固定金额（不参与通胀调整），若为一次性则在第 1 年计入
  - 负债支出：若有负债，将负债按剩余年限均摊（或按输入的年还款额计入）
  - **当年净现金流** = 当年总收入 − 当年总支出
  - **年末净资产** = 年初净资产 × (1 + 资产收益率) + 当年净现金流

**Step 3 — 判定结果**
- 若所有年份的 **年末净资产 ≥ 0** → **财务自由达成 ✓**
- 否则 → **财务自由未达成 ✗**，显示：
  - **最大资金缺口** = min(所有年份年末净资产) （即最困难的一年差多少钱）
  - **缺口出现年份**

**Step 4 — 汇总输出表**
- 生成年度明细表，包含年份、年龄（各人）、当年收入、当年支出、当年净现金流、年末净资产等字段
- 生成汇总统计：总收入、总支出、总净现金流、初始净资产、最终净资产

### 3.3 显示模块

界面结构如下：

#### 3.3.1 侧边栏 — 输入面板
所有输入表单放在 Streamlit 侧边栏（`st.sidebar`），按上述分组用 `st.expander` 折叠组织，节省主界面空间。

#### 3.3.2 主界面

**上半部分：结果概览卡**
- 财务自由判定结果（大号醒目文字：✅ 财务自由 或 ❌ 未达成）
- 若未达成：显示 **还差多少钱**、**缺口年份**
- 关键摘要：初始净资产、模拟年限、总收入、总支出、最终净资产

**中部：图表**

使用 Plotly 绘制：
1. **净资产趋势折线图** — 每年年末净资产变化
2. **年度收支对比柱状图** — 每年收入 vs 支出
3. **收支构成饼图** — 总收入/总支出的各来源/去向占比（可选）

**下半部分：年度明细表**
- 使用 `st.dataframe` 或 `st.table` 展示逐年模拟数据表
- 表头包含：年份、你的年龄、配偶年龄、收入合计、支出合计、净现金流、年末净资产

#### 3.3.3 底部操作区（侧边栏）

- **🔄 恢复默认值** — 将所有参数重置为出厂默认值，同时清空已缓存的计算结果并自动重算
- **🔢 重新计算** — 手动触发重新模拟
- **💾 保存参数** — 将当前全部输入参数导出为 JSON 文件供下载归档
- **📂 加载参数** — 上传之前保存的参数 JSON 文件恢复输入
- **💾 保存结果**（主界面） — 将输入数据和计算结果一起保存为 JSON 文件
- **📥 导出 CSV/Excel**（主界面） — 将年度明细表导出为表格文件
- **📂 加载结果**（主界面） — 上传之前保存的结果 JSON 文件恢复数据和图表

```json
    "params": { "inflation_rate": 0.02, "asset_return_rate": 0.02 }
  },
  "result": {
    "is_financially_free": false,
    "shortfall": 350000,
    "shortfall_year": 2045,
    "summary": { ... },
    "yearly_table": [ ... ]
  }

```

#### 3.4.2 导出明细 (CSV/Excel)
列：年份，你的年龄，配偶年龄，工资收入，房租收入，分红收入，其他收入，收入合计，生活支出，教育支出，意外支出，负债支出，支出合计，净现金流，年初净资产，年末净资产

#### 3.4.3 输入参数持久化

每次计算完成后，程序自动将当前所有输入数据写入 `%USERPROFILE%\.finfreedom_config.json`。下次启动应用时自动加载并预填表单。

```
启动应用 → 检测 ~/.finfreedom_config.json → 存在 → 加载参数 → 预填表单
                                            → 不存在 → 使用出厂默认值
计算完成 → 自动保存到 ~/.finfreedom_config.json
```

- 文件存储在用户主目录下，不同用户互不干扰
- 含 `version` 字段用于未来数据迁移
- 用户也可通过 **保存参数** / **加载参数** 按钮手动归档和恢复

## 5. 注意事项与边界情况

1. **退休后工资归零**：一旦本人/配偶年龄达到退休年龄，该人的工资收入在当年及以后年份设为 0
2. **负债处理**：负债在模拟首年从总资产中扣除作为初始净资产。若有剩余还款年限，假设负债已在过去发生，净资产中已扣除负债总额。简单模型：初始净资产 = 总资产 - 总负债
3. **小孩教育**：若小孩当前年龄已超过某个教育阶段，该阶段自动跳过；若当前正在某个阶段中，从当前年份起计算剩余年限的费用
4. **多个小孩**：每个小孩独立计算教育支出，每年所有小孩教育支出之和计入当年总支出
5. **意外预留一次性扣除**：若选择一次性扣除，第一年支出中增加意外预留全额
6. **资产收益率**：资产收益率作用于年初净资产，模拟理财/利息收入
7. **通胀计算**：所有收入和支出（除了意外预留均摊部分和固定赞助费）均按通胀率逐年复利增长
8. **年龄计算**：每年所有人年龄 +1，直到达到预期寿命

## 7. 打包发布（Windows）

### 7.1 技术方案

使用 **PyInstaller `onedir` 模式** 将 Python + Streamlit 应用打包为可执行目录。

| 组件 | 说明 |
|------|------|
| 入口文件 | `launcher.py` — 设置环境变量、固定端口 3568、启动后自动打开浏览器 |
| 打包配置 | `FinFreedom.spec` — 含 `collect_data_files('streamlit')`、`collect_submodules('streamlit')`、`copy_metadata` 等关键配置 |
| 端口 | 3568（避免与其他服务冲突） |
| 模式 | `onedir`（文件解压到 `_internal/`，Streamlit 可直接读取 `app.py`） |

### 7.2 输出结构

```
dist/财务自由评估/
├── 财务自由评估.exe        ← 双击运行
└── _internal/
    ├── app.py
    ├── modules/
    ├── models/
    └── utils/
```

### 7.3 重新打包

修改源码后：

```bash
python -m PyInstaller --clean --noconfirm FinFreedom.spec
```

或直接双击 `重打包.bat`。

见 `package-exe.md` 了解完整打包经验总结。
